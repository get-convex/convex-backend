Nos encantaría que disfrutaras *sin límites* al crear con Convex, pero las
consideraciones prácticas de ingeniería imponen algunos límites. Esta página
describe los límites actuales en el ecosistema de Convex.

Muchos de estos límites se irán relajando con el tiempo. Ponte en
contacto con nosotros si alguno resulta prohibitivo para tu aplicación.

Los límites se aplican por equipo, a menos que se indique lo contrario.

## Documentos \{#documents\}

Se aplica por documento y a cualquier `Object` anidado, a menos que se indique lo contrario.

|                     |               | Notas                                                            |
| ------------------- | ------------- | ---------------------------------------------------------------- |
| Tamaño              | 1 MiB         |                                                                  |
| Campos              | 1024          | Número de campos/claves                                          |
| Longitud del nombre de campo   | 64 caracteres | Las claves de los `Object` anidados pueden tener una longitud de hasta 1024 caracteres. |
| Profundidad de anidamiento de campos | 16            | Cuántas veces se pueden anidar objetos y arrays, por ejemplo `[[[[]]]]` |
| Elementos de array  | 8192          |                                                                  |

### Restricciones \{#restrictions\}

* Los nombres de los campos solo pueden contener caracteres ASCII alfanuméricos no de control y
  guiones bajos, y deben comenzar con un carácter alfabético o un guion bajo.
* Los documentos no pueden contener campos de primer nivel que comiencen con un guion bajo, excepto
  los campos del sistema `_id` y `_creationTime`.
* Las cadenas de texto deben ser secuencias Unicode válidas sin sustitutos (surrogates) no emparejados.

## Ejecuciones concurrentes de funciones \{#concurrent-function-executions\}

Número de funciones que puedes ejecutar al mismo tiempo para cada tipo de función.

|                        | Free/Starter | Professional<sup>*</sup> |
| ---------------------- | ------------ | ------------------------- |
| Consultas              | 16           | 256                       |
| Mutaciones             | 16           | 256                       |
| Acciones de tiempo de ejecución de Convex | 64           | 256                       |
| Acciones de Node       | 64           | 1000                      |
| Acciones HTTP          | 16           | 128                       |
| Tareas programadas     | 10           | 300                       |

<sup>*</sup> Los límites se pueden eliminar para clientes del plan Professional de forma
individual, caso por caso. Envíanos un mensaje de soporte a través del panel de control para comentar
tus necesidades. Normalmente esto solo es necesario si tu producto tiene un tráfico con picos
muy marcados.

Si alguno de estos límites no se ajusta a tus necesidades,
[háznoslo saber](https://convex.dev/community)!

Consulta nuestra [página de planes y precios](https://www.convex.dev/pricing) para
ver los límites de recursos. Una vez que se alcancen estos límites en el plan gratuito,
las nuevas mutaciones que intenten aplicar más inserciones o actualizaciones pueden fallar.
Los planes de pago no tienen límites estrictos de recursos: pueden escalar hasta miles de millones
de documentos y TB de almacenamiento.
