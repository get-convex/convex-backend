暂停部署是一种在不删除任何数据的情况下将部署“关掉”的方式。如果你有某个操作正在迅速耗尽第三方 API 配额，而你只需要一个大大的红色停止按钮，这会非常有用。

当部署被暂停时：

- 新的函数调用会返回错误。
- 计划任务会排队，并在部署恢复后运行。
- Cron 任务会被跳过。
- 其他一切（例如代码推送、仪表盘中的编辑）应照常工作。

**这点很重要！** 当部署被暂停时，所有新的函数调用都会返回错误，因此如果你在生产环境中运行应用，你可能需要考虑其他替代方案，比如推送一段代码来禁用你想“关掉”的某个功能。我们建议先在开发环境（dev）部署中测试这一功能，再暂停生产环境部署。
