## 保证 \{#guarantees\}

Convex 的正式服务条款、隐私政策和客户协议
[列在我们的正式条款中](https://www.convex.dev/legal/tos)。目前我们尚未提供超出正式条款所列内容的合同约定，本文件中的讨论不构成对这些条款的任何修订。

Convex 始终在持续演进中，未来版本可能需要你修改代码才能升级到新版本。在 Convex 1.0 或之后版本上开发的代码将继续按现状正常运行。如果我们在未来确实需要做出不兼容变更，我们会直接联系相关团队，并提前充分告知。

存储在 Convex 中的所有用户数据在静态存储时均已加密。数据库状态会在多个物理可用区之间进行可靠复制。系统会定期执行数据库备份（包括周期性备份和增量备份），并以 99.999999999%（11 个 9）的持久性进行保存。

我们以 99.99%（4 个 9）的可用性作为 Convex 部署的目标，尽管这些部署可能会在未提前通知的情况下因维护而发生停机。物理层面的故障可能会影响某个部署的可用性，但不会影响存储在 Convex 中数据的持久性。

在本文档中标记为 **beta** 的特性仍处于开发中。它们可以在生产环境中使用，但其 API 将来可能会发生变化，升级到新版 Convex npm 包或其他 Convex 客户端库时，可能需要额外的迁移工作。

## 未来功能 \{#future-features\}

Convex 仍在非常积极地开发中，这里列出了一些我们已经纳入规划但尚未实现的功能。欢迎你在 [Convex Discord 社区](https://convex.dev/community) 中分享更多你的需求。

### 授权 \{#authorization\}

Convex 当前提供一个[*身份验证框架*](/auth.mdx)，用于验证用户身份。未来我们计划加入一个&#95;授权框架&#95;，让开发者可以定义用户可以访问哪些数据。

目前，你可以在查询和变更函数中手动进行授权检查；敬请期待，我们将在未来提供一个更全面、更健壮的解决方案。

### 遥测 \{#telemetry\}

目前，仪表盘只提供基础指标。真正大规模运行的网站需要把我们的日志和指标集成进更加完善的可观测性系统中，用于对它们进行分类，并支持告警等功能。

Convex 未来将提供方法，以第三方系统可以摄取的格式发布部署数据。

### 分析 / OLAP \{#analytics-olap\}

Convex 主要用于满足你应用的所有实时事务处理（OLTP）需求。对于满足分析（OLAP）场景所需的那类复杂查询和大规模表扫描，它并不十分适合作为理想的解决方案。

Convex 提供
[Fivetran 和 Airbyte 连接器](/production/integrations/streaming-import-export.md)
以将 Convex 数据导出到外部分析系统。

### 浏览器支持 \{#browser-support\}

Convex 目前尚未制定官方的浏览器支持策略，但我们会尽力支持大多数具有较高
[使用率](https://caniuse.com/usage-table) 的现代浏览器。
