使用 Convex 时，你不需要编写任何 `CREATE TABLE` 语句，也不需要一开始就反复推敲存储表结构、字段命名和类型。你只管把对象写入 Convex，继续构建你的应用即可！

不过，前期快速推进在后期可能会带来问题。“那个字段到底是数字还是字符串？我记得在修那个 bug 的时候改过？”

过于宽松的存储系统在你的系统逐渐成熟时，有时会演变成负担，因为这时你会希望能够笃定地推理系统中到底保存了什么数据。

好消息是，Convex 始终是有类型的，只不过类型是隐式的！当你向 Convex 提交文档时，它会跟踪文档中所有字段的类型。你可以前往[仪表盘](/dashboard.md)，查看任意表推断出的模式，了解你最终实际存储了什么。

“那我把某个字段从字符串改成数字又如何？”Convex 同样可以处理这种情况。Convex 会跟踪这些变更，在这种情况下，该字段就是一个联合类型，比如 `v.union(v.number(), v.string())`。这样一来，即使你改变了对文档字段和类型的想法，Convex 也会为你兜底。

当你准备好将模式正式化时，可以使用我们的[模式构建器](/database/schemas.mdx)来定义它，以启用模式校验并基于该模式生成类型。
