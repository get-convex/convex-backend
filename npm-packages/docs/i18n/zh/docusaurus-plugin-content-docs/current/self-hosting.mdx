如果你对自托管很感兴趣，你可以在自己的服务器上运行 Convex 后端。自托管的 Convex 使用
[开源后端](https://github.com/get-convex/convex-backend)，运行的代码与云服务完全一致并保持最新。

要开始自托管，请按照自托管指南进行操作：

<CardLink
  className="convex-hero-card"
  item={{
  href: "https://github.com/get-convex/convex-backend/blob/main/self-hosted/README.md",
  label: "自托管指南",
}}
/>

加入 `#self-hosted` 频道，在
[Discord 社区](https://convex.dev/community) 中获取自托管支持。

自托管并不适合所有人。如果你在寻找更加省心的解决方案，我们建议使用
[Convex 托管服务](https://convex.dev/pricing)。

## 开源 Convex 后端 \{#open-source-convex-backend\}

后端的大部分代码是用 Rust 编写的，并辅以大量 TypeScript 来支持服务端函数执行环境。

你可以在我们的
[公告](https://news.convex.dev/convex-goes-open-source/) 和
[Software Engineering Daily 播客](https://softwareengineeringdaily.com/2024/03/20/going-open-source-at-convex-with-james-cowling/)
中进一步了解 Convex 开源的更多信息。
该项目以 [FSL Apache 2.0 许可证](https://fsl.software/) 发布。

## 其他 Convex 开源项目 \{#other-convex-open-source-projects\}

Convex 的后端、客户端库、仪表盘和 CLI 都是开源的。
你可以在
[Convex GitHub 页面](https://github.com/get-convex)
上浏览全部项目。

### Convex 客户端 \{#convex-clients\}

所有 Convex 客户端都是开源的。

* [Convex JavaScript/TypeScript 客户端和 CLI 工具](https://github.com/get-convex/convex-js)
* [Convex Python 客户端](https://github.com/get-convex/convex-py)
* [Convex Rust 客户端](https://github.com/get-convex/convex-rs)

### 还有更多 \{#much-much-more\}

Convex 还开源了许多其他实用项目，包括
[helpers](https://github.com/get-convex/convex-helpers)、[templates](https://github.com/orgs/get-convex/repositories?type=all\&q=template)、[demos](https://github.com/get-convex/convex-demos)、一个
[testing harness](https://github.com/get-convex/convex-test) 等等。请在
[GitHub 上](https://github.com/orgs/get-convex/repositories?type=all)查看我们所有公开仓库的完整列表。
