私たちは、Convex 上での開発を思う存分楽しんでいただきたいと考えていますが、
エンジニアリング上の実務的な理由から、いくつかの制限を設けています。
このページでは、Convex エコシステムにおける現在の制限について説明します。

これらの制限の多くは、時間の経過とともにより緩和されていく予定です。
もしアプリケーションにとって支障となる制限があれば、ぜひご連絡ください。

特にことわりがない限り、制限はチームごとに適用されます。

## ドキュメント \{#documents\}

特に明記がない限り、各ドキュメントおよびネストされたすべての `Object` に適用されます。

|                     |               | 備考                                                                 |
| ------------------- | ------------- | -------------------------------------------------------------------- |
| サイズ              | 1 MiB         |                                                                      |
| フィールド数        | 1024          | フィールド／キーの数                                                 |
| フィールド名の長さ  | 64 文字       | ネストされた `Object` のキーは最大 1024 文字まで使用できます。       |
| フィールドのネスト深さ | 16          | オブジェクトと配列をどの程度ネストできるか。例: `[[[[]]]]`          |
| 配列要素数          | 8192          |                                                                      |

### 制約 \{#restrictions\}

* フィールド名には、制御文字以外の ASCII 英数字とアンダースコアのみを使用でき、先頭は英字またはアンダースコアでなければなりません。
* ドキュメントには、システムが提供する `_id` と `_creationTime` フィールドを除き、先頭がアンダースコアのトップレベルフィールドを含めることはできません。
* 文字列は、対になっていないサロゲートを含まない、有効な Unicode シーケンスでなければなりません。

## 同時関数実行数 \{#concurrent-function-executions\}

各関数タイプごとに同時に実行できる関数の数です。

|                        | Free/Starter | Professional<sup>*</sup> |
| ---------------------- | ------------ | ------------------------- |
| クエリ                 | 16           | 256                       |
| ミューテーション       | 16           | 256                       |
| Convex ランタイム アクション | 64      | 256                       |
| Node アクション        | 64           | 1000                      |
| HTTP アクション        | 16           | 128                       |
| スケジュール済みジョブ | 10           | 300                       |

<sup>*</sup> Professional プランをご利用のお客様については、個別に制限を引き上げることができます。
ニーズについて相談したい場合は、ダッシュボードからサポートメッセージを送信してご相談ください。
通常、これはプロダクトのトラフィックに非常に大きなスパイクがある場合にのみ必要です。

[お知らせください](https://convex.dev/community)。

リソース上限については、[プランと料金ページ](https://www.convex.dev/pricing)を参照してください。無料プランでこれらの上限に達した後は、さらに挿入や更新をコミットしようとする新しいミューテーションは失敗する可能性があります。有料プランには厳格なリソース上限はなく、数十億件のドキュメントと TB 単位のストレージまでスケールできます。
