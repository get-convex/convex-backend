## 保証事項 \{#guarantees\}

Convex の正式な利用規約、プライバシーポリシー、およびカスタマー契約は
[公式利用規約ページ](https://www.convex.dev/legal/tos)に記載されています。現時点では、
この公式利用規約に記載されている内容以外の契約上の取り決めはなく、
本ドキュメント内での議論はこれらの規約を変更・修正するものではありません。

Convex は常に継続的に開発されており、将来のリリースでは新しいバージョンにアップグレードするために
コードの変更が必要になる場合があります。Convex 1.0 以降で開発されたコードは、そのまま動作し続けます。
将来、破壊的変更を行う必要が生じた場合には、十分な事前通知を行うため、該当するチームに直接ご連絡します。

Convex に保存されるすべてのユーザーデータは、保存データが暗号化されています。
データベースの状態は、複数の物理的なアベイラビリティゾーンにわたって耐久的にレプリケーションされています。
定期的かつ増分的なデータベースバックアップが実行され、99.999999999%（9 が 11 個）の耐久性で保存されます。

Convex のデプロイメントについては 99.99%（9 が 4 個）の可用性を目標としていますが、
事前の通知なしにメンテナンスによるダウンタイムが発生する場合があります。
物理的な障害はデプロイメントの可用性に影響を与える可能性がありますが、
Convex に保存されているデータの耐久性には影響しません。

## ベータ機能 \{#beta-features\}

これらのドキュメントで **beta** とタグ付けされている機能は、まだ開発中です。本番環境でも利用できますが、将来的に API が変更される可能性があり、その場合は Convex の npm パッケージや他の Convex クライアントライブラリを新しいバージョンにアップグレードする際に、追加の作業が必要になることがあります。

## 今後の機能 \{#future-features\}

Convex は現在も非常に活発に開発が進められており、ここでは今後対応を予定している未実装の機能を挙げています。あなたの要件についても、ぜひ [Convex Discord Community](https://convex.dev/community) で教えてください。

### 認可 \{#authorization\}

Convex には現在、ユーザーのアイデンティティを検証するための[*認証フレームワーク*](/auth.mdx)があります。将来的には、開発者がユーザーごとにアクセス可能なデータを定義できるようにする、&#95;認可フレームワーク&#95;を追加する予定です。

現時点では、クエリやミューテーション内で手動の認可チェックを実装する必要がありますが、将来的にはより包括的で堅牢なソリューションを提供する予定なのでご期待ください。

### テレメトリー \{#telemetry\}

現在、ダッシュボードで提供されているのは基本的なメトリクスだけです。大規模に運用されるサービスでは、
ログやメトリクスを、分類やアラートなどを可能にする、より本格的な
オブザーバビリティシステムに統合する必要があります。

Convex は将来的に、サードパーティによって取り込める形式でデプロイメントデータを
公開するためのメソッドを提供する予定です。

### 分析 / OLAP \{#analytics-olap\}

Convex は、アプリケーションにおけるリアルタイムな実装
(OLTP) のニーズを主に満たすように設計されています。分析 (OLAP) のユースケースの要件に対応するために必要となるような、複雑なクエリや大規模なテーブルスキャンにはあまり適していません。

Convex は
[Fivetran と Airbyte コネクタ](/production/integrations/streaming-import-export.md)
を提供しており、Convex のデータを外部の分析システムへエクスポートできます。

### ブラウザサポート \{#browser-support\}

Convex ではまだ公式なブラウザサポートポリシーはありませんが、十分な
[利用シェア](https://caniuse.com/usage-table)
があるほとんどのモダンブラウザをサポートできるよう努めています。
