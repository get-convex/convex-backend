デプロイメントを一時停止すると、データを一切削除せずにデプロイメントを「停止」できます。これは、あるアクションがサードパーティの API のクォータを消費し続けてしまっていて、とにかく大きな赤いストップボタンが欲しいときに便利です。

デプロイメントが一時停止されているときは:

- 新しい関数呼び出しはエラーを返します。
- スケジュールされたジョブはキューされ、デプロイメントが再開されたときに実行されます。
- cron ジョブはスキップされます。
- それ以外（例: コードのプッシュやダッシュボードでの編集）は通常どおり動作するはずです。

**ここが重要です！** デプロイメントが一時停止されている間は、すべての新しい関数呼び出しがエラーを返すため、本番環境でアプリを運用している場合は、「停止」したい機能を無効化するコードをプッシュするなど、別の方法を検討したほうがよいかもしれません。まず dev デプロイメントでこの機能をテストしてから、本番デプロイメントを一時停止することを推奨します。
