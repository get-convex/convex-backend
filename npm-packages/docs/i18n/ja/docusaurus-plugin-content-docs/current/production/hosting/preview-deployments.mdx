Convex のプレビューデプロイメントを使用すると、本番環境に反映する前に
バックエンドの変更をチームでテストできます。

Vercel の Preview Deployments や Netlify の Deploy Previews と組み合わせることで、
フロントエンドとバックエンドの変更をまとめてプレビューできます。

<BetaAdmonition feature="Convex preview deployments" verb="are" />

## セットアップ \{#setup\}

フロントエンドとバックエンドのプレビューをまとめてセットアップする方法や、
Convex のプレビューデプロイメントの仕組みの詳細については、
[Vercel](/production/hosting/vercel.mdx#preview-deployments) または
[Netlify](/production/hosting/netlify.mdx#deploy-previews) のホスティングガイドに従ってください。

`npx convex deploy` で利用可能なすべてのオプションは、`npx convex deploy --help` を参照してください。

## 制限事項 \{#limits\}

Convex のプレビューデプロイメントは、作成から 5 日後（Professional Convex プランでは 14 日後）、または同じ名前の新しいプレビューデプロイメントが作成されたときに自動的にクリーンアップされます。Convex のダッシュボードから手動で削除することもできます。

プレビューデプロイメントも、デプロイメント数の上限に含まれます。

Convex のプレビューデプロイメントが削除されると、Vercel/Netlify のプレビューリンクは開いて UI を表示しますが、存在しない Convex デプロイメントを参照しているため、どの Convex 関数も実行できません。このような場合は、Vercel/Netlify で再デプロイすることで、新しい Convex デプロイメントを指すリンクが生成されます。

Convex では、特定の名前（Git ブランチ）ごとに同時に 1 つのデプロイメントのみが許可されます。既存ブランチに更新をプッシュすると、そのデプロイメントは削除され、新しい Convex デプロイメントに置き換えられるまでの間、プレビューリンクは Convex 関数を実行できない状態になります。

Convex のプレビューデプロイメントには、関数を実行することで初期データをセットアップできます。現時点では、他にデータをセットアップする方法はありません。つまり、本番データのコピーに対する変更の確認や、別の Convex デプロイメントからのデータのインポートには対応していません。

関数呼び出しが失敗した場合、`deploy` コマンドも失敗しますが、新しいプレビューデプロイメント自体はすでにプロビジョニング済みである点に注意してください。最善の対処は、関数内の問題を修正して再デプロイすることです。

プレビューデプロイメントのサポートはベータ機能です。[Discord でフィードバック](https://convex.dev/community) をお寄せください。
