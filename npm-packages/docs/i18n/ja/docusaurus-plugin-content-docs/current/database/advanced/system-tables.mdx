  "スケジュールされた関数やファイルストレージ情報など、Convex の組み込み機能のメタデータにシステムテーブルを通じてアクセスします。"
---

システムテーブルを使うと、Convex の組み込み機能のメタデータに読み取り専用でアクセスできます。
現在、公開されているシステムテーブルは 2 つあります:

- `"_scheduled_functions"` テーブルには、
  [スケジュールされた関数](/scheduling/scheduled-functions.mdx#retrieving-scheduled-function-status) のメタデータが含まれます
- `"_storage"` テーブルには、
  [保存されたファイル](/file-storage/file-metadata.mdx) のメタデータが含まれます

`db.system.get` と `db.system.query` メソッドを使用して、システムテーブルからデータを読み取ることができます。これらは標準の `db.get` および
`db.query` メソッドと同様に動作します。システムテーブルを読み取るクエリも、他のすべてのテーブルを読み取るクエリと同様にリアクティブかつリアルタイムであり、単一のクエリで読み切れないほど多くのドキュメントがある場合でも、ページネーションを使ってすべてのドキュメントを列挙できます。
