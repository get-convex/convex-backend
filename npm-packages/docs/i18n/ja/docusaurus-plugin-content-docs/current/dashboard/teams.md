Convex では、プロジェクトはチームごとに整理されています。チームは、
他の人とプロジェクトへのアクセスを共有するために利用します。Convex
ダッシュボード上部に表示されているチーム名をクリックすると、チームを
切り替えたり新しいチームを作成したりできます。これによりプロジェクトセレクター
が開き、そこで再度チーム名をクリックするとチームを切り替えられます。

![Team switcher](/screenshots/team_selector.png)

プロジェクト一覧ページ上部にある &quot;Team Settings&quot; ボタンをクリックすると、
チーム名を変更したり、新しいメンバーをチームに招待したりできます。

## 一般 \{#general\}

[一般設定ページ](https://dashboard.convex.dev/team/settings)では、
チーム名とスラッグを変更できます。

このページからチームを削除することもできます。チームを削除するには、
そのチームのすべてのプロジェクトを削除し、他のチームメンバーをすべて
チームから外しておく必要があります。チームを削除すると、Convex のサブスクリプションは
自動的にキャンセルされます。

![チームの一般設定ページ](/screenshots/teams_general.png)

## チームメンバー \{#team-members\}

[メンバー設定](https://dashboard.convex.dev/team/settings/members) ページを使用して、チームのメンバーを招待したり削除したりできます。

![チームメンバーのページ](/screenshots/teams_members.png)

### ロールと権限 \{#roles-and-permissions\}

Convex には、チーム、プロジェクト、デプロイメントへのアクセスを管理するための 2 つのレベルのアクセス制御があります。チームレベルのロールは、ユーザーがチーム内で行える操作を制御し、プロジェクトレベルの権限は、特定のプロジェクト内で行える操作を制御します。

#### チームのロール \{#team-roles\}

チームメンバーには次のいずれかのロールを割り当てることができます:

* Admin
* Developer

チームの作成者には、自動的に Admin ロールが割り当てられます。新しいチームメンバーを招待する際に、そのメンバーのロールを選択できます。チームメンバーのロールはいつでも変更できます。

Developer ロールのメンバーは次のことができます:

* 新しいプロジェクトとデプロイメントを作成できます。新しいプロジェクトが作成されると、
  そのプロジェクトの作成者には自動的にそのプロジェクトの
  [Project Admin](#project-admins) ロールが付与されます。
* 既存のプロジェクトを表示し、それらのプロジェクトに対する開発用およびプレビュー用デプロイメントを作成できます。Developer ロールのメンバーは本番デプロイメントからデータを読み取ることはできますが、
  書き込むことはできません。
* チームの利用状況および請求状況（過去および今後の請求書など）を表示できます。

Admin ロールのメンバーは Developer ロールのメンバーができることに加えて、次のことができます:

* 新しいチームメンバーを招待する
* チームからメンバーを削除する
* 他のチームメンバーのロールを変更する
* チームの Convex サブスクリプションと請求の詳細を管理する
* チーム名とスラッグを変更する
* Team Admin は、チーム内のすべてのプロジェクトに対する Project Admin アクセスも暗黙的に付与されます。詳細については [Project Admins](#project-admins) を参照してください。

#### プロジェクト管理者 \{#project-admins\}

チームロールに加えて、チームメンバーに「Project Admin」ロールを付与することで、
個々のプロジェクトに対する管理者アクセス権も付与できます。

特定のプロジェクトの Project Admin である場合、次のことができます：

* プロジェクト名とスラグを更新する
* プロジェクトのデフォルト環境変数を更新する
* プロジェクトを削除する
* 本番デプロイメントに書き込みを行う

メンバー設定ページで、複数のプロジェクトに対する Project Admin ロールを同時に
割り当てたり削除したりできます。複数のメンバーに対して Project Admin ロールを同時に
割り当てまたは削除するには、代わりに
[Project Settings](/dashboard/projects.md#project-settings) ページを開いてください。

## 請求 \{#billing\}

[Billing ページ](https://dashboard.convex.dev/team/settings/billing)を使用して、
Convex サブスクリプションを上位のティアにアップグレードしたり、既存の
サブスクリプションを管理したりできます。

有料プランでは、請求担当者情報や支払い方法を更新したり、請求書を
表示したりすることもできます。

[Convex の料金について詳しく見る](https://www.convex.dev/pricing)。

![チームの請求ページ](/screenshots/teams_billing.png)

### 支出上限 \{#spending-limits\}

有効な Convex サブスクリプションがある場合、チームの支出上限を
[請求ページ](https://dashboard.convex.dev/team/settings/billing)で設定できます。

* **警告しきい値** はソフトリミットです。この値を超えても、
  チームにはメールで通知が送信されるだけで、他の処理は行われません。
* **無効化しきい値** はハードリミットです。この値を超えると、
  チーム内のすべてのプロジェクトが無効化されます。これにより、プロジェクト内で
  関数を実行しようとするとエラーがスローされます。上限を引き上げるか削除することで、
  プロジェクトを再度有効化できます。

支出上限は、プランに含まれている量を超えてチームのプロジェクトが使用した
リソースにのみ適用されます。シート料金（チーム内の各開発者ごとに支払う金額）は
上限には含まれません。例えば、支出上限を $0/月 に設定した場合は、
シート料金のみが課金され、プランに組み込まれているリソースの範囲を超えると
プロジェクトは無効化されます。

![支出上限が設定されたチームの請求ページ。](/screenshots/teams_billing_spending_limits.png)

## 使用状況 \{#usage\}

[使用状況ページ](https://dashboard.convex.dev/team/settings/usage)では、
チームが利用したすべてのリソースと、プランの上限に対してどの程度利用しているかを確認できます。

[Convex の料金について詳しく見る](https://www.convex.dev/pricing)。

![チームの使用状況ページ](/screenshots/teams_usage.png)

すべてのメトリクスは日別の内訳で確認できます。

![チームの使用状況ページのグラフ](/screenshots/teams_usage_2.png)

## 監査ログ \{#audit-log\}

<Admonition type="info">
  監査ログは Convex Professional でのみ利用できます。
</Admonition>

[監査ログページ](https://dashboard.convex.dev/team/settings/audit-log)では、
チーム内のメンバーによって行われたすべての操作を確認できます。これには
プロジェクトおよびデプロイメントの作成と管理、チームメンバーの招待および削除、
その他の操作が含まれます。

![チーム監査ログページ](/screenshots/teams_audit_log.png)

[デプロイメント履歴ページ](/dashboard/deployments/history.md)では、
デプロイメントに関連するイベントの履歴も表示できます。
